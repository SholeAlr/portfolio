# JavaScript Closures Explained Without Making Your Brain Cry ☕

If you've ever Googled **"JavaScript closures"**, you've probably seen this definition:

> **A closure is the combination of a function bundled together with references to its surrounding lexical environment.**

Cool.

I understood every word individually.

Just... not the sentence.

So let's forget the scary wording and learn closures like normal humans.

---

## Imagine a Coffee Shop ☕

You're at your favorite coffee shop.

You order a cappuccino.

The barista writes your name on the cup.

Five minutes later...

> **"Sholeh! Cappuccino!"**

How did they remember your name?

Because the cup kept the information.

Closures work the same way.

When JavaScript creates a function, it secretly gives it a tiny backpack 🎒.

Inside that backpack are all the variables the function might need later.

Even if the outer function disappears...

The backpack stays.

---

## First Example

```js
function makeGreeting(name) {
  return function () {
    console.log(`Hello ${name}!`);
  };
}

const greet = makeGreeting("Sholeh");

greet();
```

Output:

```text
Hello Sholeh!
```

Wait...

`makeGreeting()` already finished.

Shouldn't `name` disappear?

Normally...

Yes.

But because the returned function still needs `name`, JavaScript keeps it alive.

That's a closure.

---

## Let's Slow Down

When this runs:

```js
const greet = makeGreeting("Sholeh");
```

JavaScript creates:

```text
name = "Sholeh"
```

and returns:

```js
function () {
  console.log(name);
}
```

Instead of throwing `name` away, JavaScript says:

> "This function still needs it."

So it keeps it inside the function's little backpack.

```
Function
│
├── Code
└── Backpack 🎒
      └── name = "Sholeh"
```

Later...

```js
greet();
```

The function opens its backpack and finds:

```
🎒

name = "Sholeh"
```

Then prints it.

Easy.

---

## Another Example 🍪

Imagine a cookie jar.

```js
function cookieJar() {
  let cookies = 5;

  return function () {
    cookies--;

    console.log(`${cookies} cookies left.`);
  };
}

const eatCookie = cookieJar();
```

Now call it:

```js
eatCookie();
```

Output:

```text
4 cookies left.
```

Again:

```js
eatCookie();
```

Output:

```text
3 cookies left.
```

Where is `cookies` stored?

Inside the closure.

The backpack now contains:

```
🎒

cookies = 3
```

Each call updates the value.

---

## Why Is This Useful?

Without closures...

Every function would forget everything immediately.

Closures give functions memory.

They're like your phone remembering your Wi-Fi password.

Imagine typing it every single time.

Exactly.

Nobody wants that.

---

## Real-World React Example

```jsx
const [count, setCount] = useState(0);

<button onClick={() => setCount(count + 1)}>+</button>;
```

That arrow function remembers `count`.

That's a closure.

---

Another one:

```js
setTimeout(() => {
  console.log("Hello!");
}, 1000);
```

One second later...

The callback still remembers variables from where it was created.

Closure again.

---

## Private Variables 🔒

Closures are also useful for hiding data.

```js
function bankAccount() {
  let balance = 100;

  return {
    deposit(amount) {
      balance += amount;
    },

    withdraw(amount) {
      balance -= amount;
    },

    getBalance() {
      return balance;
    },
  };
}

const account = bankAccount();

account.deposit(50);

console.log(account.getBalance());
```

Output:

```text
150
```

Can someone do this?

```js
account.balance = 999999;
```

No.

Because `balance` isn't public.

It's safely hidden inside the closure.

Kind of like my snacks when guests come over.

---

## Common Mistake

People often think:

> "The outer function finished, so all its variables are gone."

Not necessarily.

If another function still needs them...

JavaScript keeps them alive.

Only when nothing references them anymore can the garbage collector remove them.

---

## Mental Model

Whenever you see this:

```js
function outer() {
  let value = 42;

  return function inner() {
    console.log(value);
  };
}
```

Don't think:

> "Lexical environment."

Think:

> "The inner function walked away carrying a backpack."

Inside that backpack:

```
🎒

value = 42
```

That's really all a closure is.

---

## TL;DR

A closure is:

> **A function that remembers variables from where it was created, even after the outer function has finished running.**

That's it.

No magic.

No scary definitions.

Just a tiny backpack full of memories.

---

## Final Thoughts

The first time I learned closures, I felt like JavaScript was trying to gaslight me.

> "The function is gone... but somehow the variable is still there?"

Eventually it clicked.

Now, whenever I see a callback, an event handler, or a React Hook, I imagine a tiny function walking around with a backpack full of variables.

And honestly...

That's much easier to remember than **"references to its surrounding lexical environment."**
